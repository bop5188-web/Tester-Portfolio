Problem

Earlier versions looked simple and awkward — weak layout, odd button placement, thin proof, and project choices that didn’t show what I can actually do. They didn’t read as professional for a first full-time search.

What I changed

  • Rebuilt structure and visual system for a cleaner, more professional feel.
  • Cut projects that weren’t useful — weak proof, wrong skill signal, or not a fair look at my work.
  • Rewrote cases around ownership, skills, and results so recruiters can scan without a long scroll.

How I get feedback

  • Ask anyone who’ll be honest: friends, parents, my brother, and my mom (who works in this field).
  • Ship in versions — freshman Adobe → previous site → this summer’s rebuild — not one endless tweak.
  • Use critique to decide what stays: if a project doesn’t show skills or proof, it’s out (archive only).

Result

A third-pass site that matches how I want to be hired: clearer proof, better project choices, and a process that takes feedback seriously. Older work stays in the archive — linked only here.
